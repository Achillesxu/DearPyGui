Dear PyGui

Widget ID System

In Dear PyGui, all items must have an associated unique ID. This is stricter than Dear ImGui in which some widgets do not need an ID at all. The reasons for this:

  • To satisfy Dear ImGui's requirements.
  • To allow for modifications of the UI at runtime.
The ID of a widget is different than its label, however most widgets that have a label will use the ID by default. This becomes an issue when multiple widgets need to have the same label. There are 2 ways to address this, which we cover in the following sections.

Recommended Method

The recommended method for addressing collision of IDs when trying to display widgets with the same label is to directly set the label using the label keyword. For example: 

add_button("UniqueButtonID1", label="Press me")
add_button("UniqueButtonID2", label="Press me")

Alternative Method

Dear ImGui allows users to append "##" to a widget's ID with additional text that will not be visible; this can be used to keep the ID unique. Because Dear PyGui's widgets set their label to their ID by default, this will pass through to ImGui if you do not override the label. This method can be used in place of the "label" method however it is not guaranteed to work for all widgets (i.e. text widgets). For example: 

add_button("Press me##ID1")
add_button("Press me##ID2")
