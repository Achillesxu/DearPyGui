Overview

This tutorial is intended to demonstrate how to use some features of DearImGui. The examples do not showcase every ability in DearPyGUI "DPG". More complex examples are included in the Examples folder on github. It is also recommended to refer to the API Reference for the information on a specific function.

DearPyGui consists of a program window, windows, and widgets. The program window is the main window of your program and is created at the end of the main python script by calling the function start_dearpygui()

In the next example please look at the about window and verify that you have the latest DearPyGUI package installed even if you just did a pip install. some IDE's use virtual environments which can lead to multiple instances of packages being installed.

Here is an example of the program window and another window showing documentation using the Built-in documentation function: 

import dearpygui.core as dpg

dpg.set_main_window_size(800, 800)
dpg.show_about()
dpg.show_documentation()
# when running this code please look at the about window and it will report which version of DearPyGUI is running
# Even if you just did a pip install please ensure you your environment is using the latest package
dpg.start_dearpygui()

DPG is made up of two modules "dearpygui.core" and "dearpygui.simple".

"dearpygui.core" contains the base functionality of DearPyGUI. Everything can be done with core.

"dearpygui.simple" contains simple wrappers created from core to aid in cleaner and faster code. As your code complexity grows its recommended to start using simple.

Throughout the tutorial we will show each example using both methods core only and simple.

Developer Tools

DearPyGui contains several very useful developer tools which we will utilize throughout the tutorials.

The show_source() requires an input of a python file name to read. For our example our file name is "main.py" so we are using "main.py". You should replace "main.py" with your file name. 

import dearpygui.core as dpg

dpg.show_documentation()
dpg.show_debug()
dpg.show_about()
dpg.show_metrics()
dpg.show_source("main.py")  # replace"main.py" with your python file name
dpg.show_logger()

dpg.start_dearpygui()

Built-in Logging

One of the many powerful built in developer tools is the Logger. This can be accessed through the command show_logger(). The logger has 6 log levels:

  1. Trace
  2. Debug
  3. Info
  4. Warning
  5. Error
  6. Off
The log commands can be filtered by setting the log level. The logger will filter out the set level and any levels lower than that level.

"mvTRACE" log level will show all the commands.

see below for an example 

import dearpygui.core as dpg

dpg.show_logger()
dpg.set_log_level(dbp.mvTRACE)
dpg.log("trace message")
dpg.log_debug("debug message")
dpg.log_info("info message")
dpg.log_warning("warning message")
dpg.log_error("error message")

dpg.start_dearpygui()

Creating Widgets and Containers

DPG items can be broken dow into :

  • regular items (ex. inputs, buttons, and display items)
  • container items (ex. window, group, popup, tooltip, child, ect)
Items are added by their respective "add_***" commands.

Every item must have a unique name. By default the name is what is shown on the screen. In the event that you would like widgets to appear on screen with the same name you may either
  • 1. place a "##" trailing the name (ex. "displayed_name##unique_part") anything after the "##" will be hidden from the displayed name.
  • 2. use the label keyword which will display the label instead of the name for the item.
Some item names are auto generated for items without name arguments in the method. (i.e.same_line, spacing, separator, unindent, add_indent). However these have an optional name keyword which can be filled out if you need to reference the item at a later time. 

import dearpygui.core as dpg

dpg.add_button("Apply")
dpg.add_same_line(spacing=10)
dpg.add_button("Apply##1")
dpg.add_same_line(spacing=10, name="sameline1")
dpg.add_button("Apply2", label="Apply")
dpg.add_spacing(count=5, name="spacing1")
dpg.add_button("Apply##3")

dpg.start_dearpygui()

Container items are used to hold other items and are implemented by either

  • starting the container with its "add***()" function, then completing the container the "end()" function.
  • (recommended) using "dearpygui.simple" and and the corresponding containers context manager.
When container items are "added", they are pushed onto the parent stack. Any item added after will belong to the parent on the top of the stack. When a container item is "ended" they are popped off of the parent stack. 

import dearpygui.core as dpg

dpg.add_text("This is some text on the MainWindow")
dpg.add_window("window 2")
dpg.add_text("This is some text on window 2")
dpg.end()
dpg.add_text("More text on the MainWindow")

dpg.start_dearpygui()

"dearpygui.simple" automates calling the function "end()", allows for code folding and shows a hierarchy in the code itself. 

import dearpygui.core as dpg
import dearpygui.simple as simple

dpg.add_text("This is some text on the MainWindow")
with simple.window("window 2"):
    dpg.add_text("This is some text on window 2")
dpg.add_text("More text on the MainWindow")

dpg.start_dearpygui()

By default items are created in the order they are coded.

However, items can be added out of order by specifying a "parent" container and where inside of the container by specifying the "before" keyword. If "before" is not specified to be added before an item then it will be appended to the end of the container. 

import dearpygui.core as dpg

dpg.add_text("First coded widget")  # this is a display item
dpg.add_text("This is some text on window 2", parent="window 2")  # we can even specify the parent before it was coded
dpg.add_text("This is some text on child 1", parent="child 1")  # we can even specify the parent before it was coded
dpg.add_window("window 2")  # this is a group item
dpg.add_child("child 1")  # this is a child container item
dpg.add_checkbox("Checkbox")  # this is a input item added inside of the child
dpg.end()
dpg.end()
dpg.add_checkbox("Last coded widget", parent="MainWindow", before="First coded widget")
dpg.add_checkbox("Last coded widget 2", parent="child 1", before="Checkbox")
# empty window
dpg.add_window("window 3")
dpg.end()
dpg.start_dearpygui()

Now the same example but with "dearpygui.simple" 

import dearpygui.core as dpg
import dearpygui.simple as simple

dpg.add_text("First coded widget")  # this is a display item
dpg.add_text("This is some text on window 2", parent="window 2")  # we can even specify the parent before it was coded
dpg.add_text("This is some text on child 1", parent="child 1")  # we can even specify the parent before it was coded
with simple.window("window 1"):  # this is a group item
    with simple.child("child 1"):  # this is a child container item
        dpg.add_checkbox("Checkbox")  # this is a input item added inside of the child
dpg.add_checkbox("Last coded widget", parent="MainWindow", before="First coded widget")
dpg.add_checkbox("Last coded widget 2", parent="child 1", before="Checkbox")
# empty window
with simple.window("window 3"):
    pass

dpg.start_dearpygui()

Every input item has a value which can be set using the "default_value" keyword when being created. Also this can be set at runtime by the commands set_value() get_value(). This is demonstrated below.

import dearpygui.core as dpg

dpg.add_checkbox("Radio Button", default_value=False)
print("First value of the Radio Button is: ", dpg.get_value("Radio Button"))
dpg.set_value("Radio Button", True)
print("Value after setting the Radio Button is: ", dpg.get_value("Radio Button"))

dpg.start_dearpygui()

Item and Window Callbacks

Each input widget has a callback which is activated when the widget is interacted with.

Callbacks are used to give functionality to widgets. Almost every widget in DearPyGui can use a callback. Callbacks can either be assigned to widgets upon creation or after creation using set_item_callback() as shown in the code below.

Every callback function applied to an item in DearPyGui must take in a "sender" and "data" argument.

The "sender" is used by DearPyGui to inform the callback which widget triggered the callback bu sending the name.

The "data" argument is used by various standard callbacks to send additional data by specifying the callback callback_data. 

import dearpygui.core as dpg


def button_callback(sender, data):
    dpg.log_debug(f"sender is: {sender}")
    dpg.log_debug(f"data is: {data}")


dpg.show_logger()  # were going to use the logger here to show the result
dpg.add_input_text("Input Text", default_value="Hello World!")
dpg.add_button("Apply", callback=button_callback, callback_data=dpg.get_value("Input Text"))
dpg.add_button("Apply##2", tip="callback was set after item was created")
dpg.set_item_callback("Apply##2", callback=button_callback, callback_data=dpg.get_value("Input Text"))
dpg.start_dearpygui()

Window type widgets have special callbacks that are triggered on events such as window resizing and window closing. The window specific callbacks can be applied to any window type widget. "on_close" will run the callback assigned to the keyword on window close and set_resize_callback() will run on every resize of the container and can be set to any specific window by using the handler keyword, the default is the "MainWindow"

Container Type items applicable for the resize callbacks are listed below:

  • MainWindow
  • Window
  • Child
  • Popup
Callbacks can be run on every completion of the render loop by using set_render_callback()
render callbacks will run every frame. 

import dearpygui.core as dpg


def close_me(sender, data):
    dpg.log_debug(f"{sender} window has been closed")


def render_me(sender, data):
    dpg.log_debug(f"window {sender} has ran a render callback")


def resize_me(sender, data):
    dpg.log_debug(f"window {sender} has ran a rezise callback")


dpg.show_logger()  # were going to use the logger to display callback replies
dpg.add_window("Tester", on_close=close_me)
dpg.add_text('resize this window resize callback will occur')
dpg.add_text('close this window using the "x" button and a close callback will occur')
dpg.end()
dpg.set_render_callback(render_me)
dpg.set_resize_callback(resize_me, handler="Tester")

dpg.start_dearpygui()

Run-time Adding & Deleting Widgets

With DearPyGui you can dynamically add and delete any items at runtime. This can be done by using a callback method to run the desired item's "add" command and specifying the parent the item will belong to. By default the widget will be added to the MainWindow if there is not a parent specified.
By using the keyword "before" when adding a item you can control which item in the parent the new item will come before. Default will place the new widget at the end.

In this example we used the "debug" developer tool to view the current state of the widgets. Go to the "App Item Layout" tab on the debug widget and this tab displays the state and properties of the App. clicking on the "MainWindow" tree item we can see the widgets currently displayed on the MainWindow. Try clicking the "Add/Delete" buttons on the main window and see the tree state change. 

import dearpygui.core as dpg
import dearpygui.simple as simple


def add_buttons(sender, data):
    dpg.add_button("New Button")
    dpg.add_button("New Button 2", parent="Secondary Window")


def delete_buttons(sender, data):
    dpg.delete_item("New Button")
    dpg.delete_item("New Button 2")


dpg.show_debug()
dpg.add_button("Add Buttons", callback=add_buttons)
dpg.add_button("Delete Buttons", callback=delete_buttons)
with simple.window("Secondary Window"):
    pass

dpg.start_dearpygui()

When deleting a container the container plus it's children are deleted by default. Unless the keyword "children_only" is true then the container will persist. 

import dearpygui.core as dpg
import dearpygui.simple as simple


def add_widgets(sender, data):
    with simple.window("Secondary Window"):
        dpg.add_button("New Button 2")
    dpg.add_button("New Button")
    dpg.add_button("New Button 3", parent="Secondary Window")


def delete_widgets(sender, data):
    dpg.delete_item("Secondary Window")
    dpg.delete_item("New Button")


def delete_children(sender, data):
    dpg.delete_item("Secondary Window", children_only=True)


dpg.show_debug()
dpg.add_button("Add Window and Items", callback=add_widgets)
dpg.add_button("Delete Window and Children", callback=delete_widgets)
dpg.add_button("Delete Window's Children", callback=delete_children)

dpg.start_dearpygui()

Linking Item Values & Data Storage

It may become necessary for one widget to control another widgets value directly or for 2 widgets to control the same value. This can be achieved through the source value item.

When a source keyword is used, the widget's value changes are bound to a value item. This can be useful to link to any widgets to use the same value.

The value can be gotten using get_value("source name").

The value can be manually set using set_value("source name").

Typically a value source item is created with the first item that uses the keyword to name it.
Most of the time this is good enough but in the rare case where the value needs to exist before the item that uses it we can use add_value().
This is typically useful when you need to link a input3 type widget to an input 4 type widget. 

import dearpygui.core as dpg


def print_me(sender, data):
    dpg.log_debug(f"checkbox values: {dpg.get_value('value_1')}")
    dpg.log_debug(f"text values: {dpg.get_value('value 2')}")
    dpg.log_debug(f"color values: {dpg.get_value('color4')}")


def reset(sender, data):
    dpg.set_value("value_1", False)
    dpg.set_value("value 2", "Hello World!")


dpg.show_logger()
dpg.add_checkbox("Radio Button1", source="value_1")
dpg.add_checkbox("Radio Button2", source="value_1")
dpg.add_input_text("Text Input 1", source="value 2")
dpg.add_input_text("Text Input 2", source="value 2", password=True, tip="this input text has password applied")
dpg.add_button("Print source values", callback=print_me)
dpg.add_button("Reset source values", callback=reset)

# special case when linking a smaller input type widget to a larger one create the large value first
dpg.add_value("color4", (0.0, 0.0, 0.0, 0.0))
dpg.add_color_edit3("color edit 3", source="color4")
dpg.add_color_edit4("color edit 4", source="color4")

dpg.start_dearpygui()

DPG provides general data for data storage also. With add_data("name", data_object) we can pass in any python item to be stored as a data item and accessed later with get_data("name")

Remember you can store any python object in data storage even custom data types. Although we will only show a map being stored. 

import dearpygui.core as dpg


def store_data(sender, data):
    custom_data = {
        "Radio Button": dpg.get_value("Radio Button"),
        "Checkbox": dpg.get_value("Checkbox"),
        "Text Input": dpg.get_value("Text Input"),
    }
    dpg.add_data("stored_data", custom_data)


def print_data(sender, data):
    dpg.log_debug(dpg.get_data("stored_data"))


dpg.show_logger()
dpg.show_debug()
dpg.add_radio_button("Radio Button", items=["item1", "item2"])
dpg.add_checkbox("Checkbox")
dpg.add_input_text("Text Input")
dpg.add_button("Store Data", callback=store_data)
dpg.add_button("Print Data", callback=print_data)


dpg.start_dearpygui()

Menu Bar

A very important widget for GUI functionality is a Menu Bar. Menu bars are always displayed on the top of a window and have 3 main parts:

  1. The Menu Bar - The main menu ribbon
  2. Menus - Drop down menus "sub-menus"
  3. Menu Items - Items that can run callbacks (basically selectables)
  4. Any Widget - Plot, drawing, ect
The Menu Bar and Menu are both container widgets and must be concluded with the respective "end" call. Using containers from "dearpygui.simple" will automate this. Items are added to the Menu Bar from right to left. Items are added to the menus from top to bottom.

Menus can be cascaded as much as necessary.

Any widget can be added to a menu. 

import dearpygui.core as dpg


def print_me(sender, data):
    dpg.log_debug(f"Menu Item: {sender}")


dpg.show_logger()
dpg.add_menu_bar("Main Menu Bar")
dpg.add_menu("File")
dpg.add_menu_item("Save", callback=print_me)
dpg.add_menu_item("Save As", callback=print_me)
dpg.add_menu("Settings")
dpg.add_menu_item("Setting 1", callback=print_me)
dpg.add_menu_item("Setting 2", callback=print_me)
dpg.end()
dpg.end()
dpg.add_menu_item("Help", callback=print_me)
dpg.add_menu("Widget Items")
dpg.add_checkbox("Pick Me", callback=print_me)
dpg.add_button("Press Me", callback=print_me)
dpg.add_color_picker4("Color Me", callback=print_me)
dpg.end()
dpg.end()

dpg.start_dearpygui()

And again with "dearpygui.simple" 

import dearpygui.core as dpg
import dearpygui.simple as simple


def print_me(sender, data):
    dpg.log_debug(f"Menu Item: {sender}")


dpg.show_logger()
with simple.menu_bar("Main Menu Bar"):
    with simple.menu("File"):
        dpg.add_menu_item("Save", callback=print_me)
        dpg.add_menu_item("Save As", callback=print_me)
        with simple.menu("Settings"):
            dpg.add_menu_item("Setting 1", callback=print_me)
            dpg.add_menu_item("Setting 2", callback=print_me)
    dpg.add_menu_item("Help", callback=print_me)
    with simple.menu("Widget Items"):
        dpg.add_checkbox("Pick Me", callback=print_me)
        dpg.add_button("Press Me", callback=print_me)
        dpg.add_color_picker4("Color Me", callback=print_me)

dpg.start_dearpygui()

File, Directory, & Save Dialogs

Directory dialogs are used by calling select_directory_dialog() which must be given a return callback. The return callback's data argument will be filled out with the directory path & folder path as a list of strings.

Typically directory dialog are called by another widget such as a button so this is how it will be done in the example. 

import dearpygui.core as dpg


def directory_picker(sender, data):
    dpg.select_directory_dialog(callback=apply_selected_directory)


def apply_selected_directory(sender, data):
    dpg.log_debug(data)  # so we can see what is inside of data
    directory = data[0]
    folder = data[1]
    dpg.set_value("directory", directory)
    dpg.set_value("folder", folder)
    dpg.set_value("folder_path", f"{directory}\\{folder}")


dpg.show_logger()
dpg.add_button("Directory Selector", callback=directory_picker)
dpg.add_text("Directory Path: ")
dpg.add_same_line()
dpg.add_label_text("##dir", source="directory", color=[255, 0, 0])
dpg.add_text("Folder: ")
dpg.add_same_line()
dpg.add_label_text("##folder", source="folder", color=[255, 0, 0])
dpg.add_text("Folder Path: ")
dpg.add_same_line()
dpg.add_label_text("##folderpath", source="folder_path", color=[255, 0, 0])

dpg.start_dearpygui()

File dialogs are used by calling open_file_dialog() which must be given a return callback. The return callback's data argument will be filled out with the directory path & file path as a list of strings. Extensions are an optional keyword to the file dialog that will give selections to filter the displayed files in the dialog based on their extensions.

Typically file dialog are called by another widget such as a button so this is how it will be done in the example. 

import dearpygui.core as dpg


def file_picker(sender, data):
    dpg.open_file_dialog(callback=apply_selected_file, extensions=".*,.py")


def apply_selected_file(sender, data):
    dpg.log_debug(data)  # so we can see what is inside of data
    directory = data[0]
    file = data[1]
    dpg.set_value("directory", directory)
    dpg.set_value("file", file)
    dpg.set_value("file_path", f"{directory}\\{file}")


dpg.show_logger()
dpg.add_button("Directory Selector", callback=file_picker)
dpg.add_text("Directory Path: ")
dpg.add_same_line()
dpg.add_label_text("##filedir", source="directory", color=[255, 0, 0])
dpg.add_text("File: ")
dpg.add_same_line()
dpg.add_label_text("##file", source="file", color=[255, 0, 0])
dpg.add_text("File Path: ")
dpg.add_same_line()
dpg.add_label_text("##filepath", source="file_path", color=[255, 0, 0])

dpg.start_dearpygui()

Plotting

DearPyGui has simple plots and plots both of which have the ability to be dynamic.

Simple plots take in a list and plot y-axis data against the number of items in the list. These can be line graphs or histograms and are demonstrated below. 

import dearpygui.core as dpg

dpg.add_simple_plot("Simpleplot1", value=[0.3, 0.9, 0.5, 0.3], height=300)
dpg.add_simple_plot("Simpleplot2", value=[0.3, 0.9, 2.5, 8.9], overlay="Overlaying", height=180, histogram=True)

dpg.start_dearpygui()

Plots have much more functionality that simple plots. Plots use both the x and y coordinates. Plots must be created using add_plot() command and then the data can bee added as either a line series or a scatter series. Features of plots are:

  • Click & Drag: to pan the plot
  • Click & Drag on Axis: to pan the plot in one direction
  • Double Click: scales plot to data
  • Right Click & Drag: to zoom to an area
  • Double Right Click: opens settings
  • Shift + Right Click & Drag: to zoom to an area that fills a current axis
  • Scroll Mouse Wheel: zooms
  • Scroll Mouse Wheel on Axis: zooms only that axis
  • Toggle data sets on the legend to hide them


Additionally a text point can be used to place floating text on the plot. 

import dearpygui.core as dpg
from math import cos, sin

def plot_callback(sender, data):
    dpg.clear_plot("Plot")

    data1 = []
    for i in range(0, 100):
        data1.append([3.14 * i / 180, cos(3 * 3.14 * i / 180)])

    data2 = []
    for i in range(0, 100):
        data2.append([3.14 * i / 180, sin(2 * 3.14 * i / 180)])

    dpg.add_line_series("Plot", "Cos", data1, weight=2, color=[0, 0, 255, 100])
    dpg.add_shade_series("Plot", "Cos", data1, weight=2, fill=[255, 0, 0, 100])
    dpg.add_scatter_series("Plot", "Sin", data2, color=[0, 255, 0, 100])


dpg.add_button("Plot data", callback=plot_callback)
dpg.add_plot("Plot", height=-1)


dpg.start_dearpygui()

Simple plots can be made dynamic by changing the value of the plot call with set_value(). This is shown below. 

import dearpygui.core as dpg
from math import sin


def on_render(sender, data):
    frame_count = dpg.get_data("frame_count")
    frame_count += 1
    dpg.add_data("frame_count", frame_count)
    plot_data = dpg.get_value("plot_data")
    if len(plot_data) > 100:
        plot_data.pop(0)
    plot_data.append(sin(frame_count/30))
    dpg.set_value("plot_data", plot_data)


dpg.add_simple_plot("Simple Plot", source="plot_data", minscale=-1.0, maxscale=1.0, height=300)
dpg.add_data("frame_count", 0)
dpg.set_render_callback(on_render)

dpg.start_dearpygui()

Plots also can be dynamic. Dynamic function can be applied as easy as clearing the plot and adding new data using a callback such as render or item's callback. set_value(). This is shown below. 

import dearpygui.core as dpg
from math import cos


def plot_callback(sender, data):
    # keeping track of frames
    frame_count = dpg.get_data("frame_count")
    frame_count += 1
    dpg.add_data("frame_count", frame_count)

    # updating plot data
    plot_data = dpg.get_data("plot_data")
    if len(plot_data) > 2000:
        frame_count = 0
        plot_data.clear()
    plot_data.append([3.14 * frame_count / 180, cos(3 * 3.14 * frame_count / 180)])
    dpg.add_data("plot_data", plot_data)

    # plotting new data
    dpg.clear_plot("Plot")
    dpg.add_line_series("Plot", "Cos", plot_data, weight=2)


dpg.add_plot("Plot", height=-1)
dpg.add_data("plot_data", [])
dpg.add_data("frame_count", 0)
dpg.set_render_callback(plot_callback)

dpg.start_dearpygui()

Drawing/Canvas

DearPyGui has a low level drawing API that is well suited for primitive drawing, custom widgets or even dynamic drawings.

The drawing is started by calling add_drawing() then items can be added by calling their respective draw commands. The origin for the canvas is in the bottom left. 

import dearpygui.core as dpg


dpg.add_drawing("Drawing_1", width=300, height=300)
dpg.draw_line("Drawing_1", [10, 10], [100, 100], [255, 0, 0, 255], 1)
dpg.draw_text("Drawing_1", [16, 16], "Origin", color=[250, 250, 250, 255], size=15)
dpg.draw_arrow("Drawing_1", [50, 70], [100, 65], [0, 200, 255], 1, 10)

dpg.start_dearpygui()

Drawings have a scale, origin, and size that can all be accessed and set. The scale is a multiplier of the items x and y values. The size is in pixels. 

import dearpygui.core as dpg
import dearpygui.simple as simple


dpg.add_drawing("Drawing_1", width=300, height=300)
dpg.draw_line("Drawing_1", [10, 10], [100, 100], [255, 0, 0, 255], 1)
dpg.draw_text("Drawing_1", [16, 16], "Origin", color=[250, 250, 250, 255], size=15)
dpg.draw_arrow("Drawing_1", [50, 70], [100, 65], [0, 200, 255], 1, 10)
simple.set_drawing_origin("Drawing_1", 150, 150)
simple.set_drawing_scale("Drawing_1", 2, 2)
simple.set_drawing_size("Drawing_1", 400, 500)

dpg.start_dearpygui()

Drawings can display images of types ".png", ".jpg", or ".bmp". Images are drawn using draw_image()

Using the keywords "pmin" and "pmax" we can define the upper left and lower right area of the rectangle that the image will be draw onto the canvas. The image will scale to fit the specified area.

With keywords "uv_min" and "uv_max" we can define a scalar number of what area on the image should be drawn to the canvas. The default of uv_min = [0,0] and uv_max = [1,1] will display the entire image while uv_min = [0,0] uv_max = [0.5,0.5] will only show the first quarter of the drawing.

To be able to demonstrate these features you must update the directory to that of an image on your computer. 

import dearpygui.core as dpg

# please update the image directory argument with a path to an image on your computer for this example
dpg.add_drawing("Drawing_1", width=700, height=700)
dpg.draw_image("Drawing_1", 'SpriteMapExample.png', [0, 700], pmax=[200, 500], uv_min=[0, 0], uv_max=[1, 1], tag="image")
dpg.draw_image("Drawing_1", 'SpriteMapExample.png', [0, 600], pmax=[200, 300], uv_min=[0, 0], uv_max=[1, 1])
dpg.draw_image("Drawing_1", 'SpriteMapExample.png', [0, 500], pmax=[200, 100], uv_min=[0, 0], uv_max=[1, 1])
dpg.draw_image("Drawing_1", 'SpriteMapExample.png', [400, 600], pmax=[600, 400], uv_min=[0, 0], uv_max=[0.5, 0.5])
dpg.draw_image("Drawing_1", 'SpriteMapExample.png', [400, 400], pmax=[700, 50], uv_min=[0, 0], uv_max=[3.5, 2.5])

Although drawings can be made dynamic by clearing and redrawing the entire thing a more efficient method is offered.

To make a drawing dynamic we should use the tag keyword to tag the items we would like to redraw. Then simply call a draw command using the same tag. This will delete only that one item and redraw it wit the new command. 

import dearpygui.core as dpg


def on_render(sender, data):
    counter = dpg.get_data("counter")
    counter += 1
    modifier = dpg.get_data("modifier")
    if counter < 300:
        modifier += 1
    elif counter < 600:
        modifier -= 1
    else:
        counter = 0
        modifier = 2

    xpos = 15 + modifier*1.25
    ypos = 15 + modifier*1.25
    color1 = 255 - modifier*.8
    color3 = 255 - modifier*.3
    color2 = 255 - modifier*.8
    radius = 15 + modifier/2
    segments = round(35-modifier/10)
    dpg.draw_circle("Drawing_1", [xpos, ypos], radius, [color1, color3, color2, 255], segments, tag="circle##dynamic")
    dpg.add_data("counter", counter)
    dpg.add_data("modifier", modifier)


dpg.add_data("counter", 0)
dpg.add_data("modifier", 2)
dpg.add_drawing("Drawing_1", width=700, height=700)
dpg.set_render_callback(on_render)

dpg.start_dearpygui()

Sprites

Using the drawing with tags, get_delta_time() and a callback ran on rendering we can create a sprite character. For this example we need to download the SpriteMapExample.png from the examples folder on github and place it next to the python file in your working directory.

The drawing is started by calling add_drawing() then items can be added by calling their respective draw commands. The origin for the canvas is in the bottom left. 

import dearpygui.core as dpg


def on_render(sender, data):

            else:
                dpg.draw_image("Drawing_1", 'SpriteMapExample.png', top_left, pmax=bottom_right, uv_min=[.8074, 0],
                           uv_max=[.8461, .10], tag="sprite")
                dpg.add_data("sprite1", True)
        else:
            if dpg.get_data("sprite1"):
                dpg.draw_image("Drawing_1", 'SpriteMapExample.png', top_left, pmax=bottom_right, uv_min=[.8464, 0],

Tables

DearPyGui has a simple table API that is well suited for static and dynamic tables.

The table widget is started by calling add_table().

To edit the table widget we can use the methods add_row() add_column() which will append the row/column to the last slot in the table.

Alternatively, we can insert rows/columns using insert_row() insert_column(). Columns and Rows are inserted according to their index argument. If the specified index already exists the exiting columns/rows will be bumped and the new row/column will be inserted at the specified index.

Also an added or inserted row/column will by default fill unspecified cells with empty cells. 

import dearpygui.core as dpg

dpg.add_table("Table Example", ["Header 0", "Header 1"])
dpg.add_row("Table Example", ["row 0", "text"])
dpg.add_row("Table Example", ["row 2", "text"])
dpg.add_column("Table Example", "Header 3", ["data","data"])
dpg.insert_row("Table Example", 1, ["row 1", "inserted row", "inserted row"])
dpg.insert_column("Table Example", 2, "Header 2", ["inserted with column", "inserted column", "inserted column"])

dpg.start_dearpygui()

Additionally, headers and cells can be renamed and their values be changed. 

import dearpygui.core as dpg


def modify_tables(sender, data):

    for coordinates in coord_list:
        dpg.set_table_item("Table Example", coordinates[0], coordinates[1], "New Value")
    dpg.set_headers("Table Example", ["New Header 0", "New Header 1", "New Header 2"])


dpg.show_logger()
dpg.add_spacing(count=5)
dpg.add_button("Modify Selected Table Values", callback=modify_tables)
dpg.add_spacing(count=5)
dpg.add_table("Table Example", ["Header 0", "Header 1"])
dpg.add_row("Table Example", ["awesome row", "text"])
dpg.add_row("Table Example", ["super unique", "unique text"])
dpg.add_column("Table Example", "Header 2", ["text from column", "text from column"])
dpg.add_row("Table Example", ["boring row"])

Input Polling

Input Polling in DearPyGui is done by calling the desired poll command within a function. The function must be set to a windows render callback for polling to occur when that window is active. Because the render callbacks are ran every frame DearPyGui can poll between frames if a specified input is performed.

A list of all the polling is as follows:

  • get_mouse_drag_delta()
  • get_mouse_pos()
  • is_key_down()
  • is_key_pressed()
  • is_key_released()
  • is_mouse_button_clicked()
  • is_mouse_button_double_clicked()
  • is_mouse_button_down()
  • is_mouse_button_dragging()
  • is_mouse_button_released()
  • set_key_down_callback()
  • set_key_press_callback()
  • set_key_release_callback()
  • set_mouse_click_callback()
  • set_mouse_double_click_callback()
  • set_mouse_down_callback()
  • set_mouse_drag_callback()
  • set_mouse_wheel_callback()
  • set_render_callback()
  • set_resize_callback()
For information about a specific command see the API Reference.

It is possible to combine as many polls as necessary to achieve desired functionality. 

import dearpygui.core as dpg


def main_callback(sender, data):
    dpg.set_value("Mouse Position", str(dpg.get_mouse_pos()))

    if dpg.is_key_down(dpg.mvKey_A):
        dpg.set_value("A key Down", "True")
    else:
        dpg.set_value("A key Down", "False")

    if dpg.is_key_pressed(dpg.mvKey_A):
        dpg.set_value("A key Pressed", "True")
    else:
        dpg.set_value("A key Pressed", "False")

    if dpg.is_key_released(dpg.mvKey_A):
        dpg.set_value("A key Released", "True")
    else:
        dpg.set_value("A key Released", "False")

    if dpg.is_mouse_button_dragging(dpg.mvMouseButton_Left, 10):
        dpg.set_value("Left Mouse Dragging", "True")
    else:
        dpg.set_value("Left Mouse Dragging", "False")

    if dpg.is_mouse_button_clicked(dpg.mvMouseButton_Left):
        dpg.set_value("Left Mouse Clicked", "True")
    else:
        dpg.set_value("Left Mouse Clicked", "False")

    if dpg.is_mouse_button_double_clicked(dpg.mvMouseButton_Left):
        dpg.set_value("Left Mouse Double Clicked", "True")
    else:
        dpg.set_value("Left Mouse Double Clicked", "False")

    if dpg.is_key_down(dpg.mvKey_Shift) and dpg.is_mouse_button_clicked(dpg.mvMouseButton_Left):
        dpg.set_value("Shift + Left Mouse Clicked", "True")
    else:
        dpg.set_value("Shift + Left Mouse Clicked", "False")

Multithreading & Asynchronous Functions

For calculations and callbacks that require extended time it can be useful to implement asynchronous functions or functions ran on separate threads. To do this simply call run_async_function().

It is important to note that functions ran with the async command cannot call any other DearPyGui API calls. 

import dearpygui.core as dpg
from time import sleep


def long_async_callback(data, sender):
    dpg.run_async_function(long_callback, None)


def long_callback(sender, data):
    sleep(3)


dpg.show_logger()
dpg.show_metrics()
dpg.add_button("long function", callback=long_callback, tip="this will cause a 3 second freeze")
dpg.add_button("long Asynchronous Function", callback=long_async_callback, tip="this will not a freeze")

dpg.start_dearpygui()

Asynchronous functions cannot access add_data() or get_data(). So when it is necessary to pass data to the asynchronous function you must use the data and return handler keyworded arguments. Any python object can be made accessible the the async function by being sent into the function through the "data" keyword. Additionally, any returned data from the Async function will be accessible through the "data" input of the specified return callback. 

import dearpygui.core as dpg
from time import sleep


def long_async_preparer(data, sender):
    floaty = dpg.get_value("Async Input Data")
    dpg.run_async_function(long_callback, floaty, return_handler=long_async_return)


def long_callback(sender, data):
    sleep(3)
    return data * 2


def long_async_return(sender, data):
    dpg.log_debug(data)


def long_callback2(sender, data):
    sleep(3)
    dpg.log_debug(data * 2)

dpg.show_logger()
dpg.add_text(
    "input a number and see the logger window for the output of the long callback that would normally freeze the GUI")
dpg.add_input_float("Async Input Data", default_value=1.0)
dpg.add_button("long Function", callback=long_callback2, callback_data=dpg.get_value("Async Input Data"), tip="This is the long callback that will freeze the gui")
dpg.add_button("long Asynchronous Function", callback=long_async_preparer, tip="this will not a freeze the GUI")

dpg.start_dearpygui()

When the asynchronous function method is called a thread pool is created. The aspects of the threadpool that can be configured are the number of threads and the timeout time.

using set_thread_count() we can set the number of threads in the thread pool. Also we can use set_threadpool_high_performance() to tell the thread pool to max out the number of threads per the users computer. Be aware that this will run the CPU at 100% when calling async functions.

Setting the timeout time for the threadpool can be done using set_threadpool_timeout(). This will destroy the threadpool and free the resources after the set amount of time.
